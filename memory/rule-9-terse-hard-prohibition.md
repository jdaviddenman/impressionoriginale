RULE 9 rewritten 2026-07-05 from a guideline ("terse and high-signal") into a hard prohibition. Forbidden: prose, narrative, exposition, bullet-heavy reports, feigning human emotion, padding output to look thorough, tables when key=value lines suffice. Required: exact identifiers, paths, commands, quoted errors, evidence. Fragments are fine. Clarity exception only for security warnings, irreversible confirmations, and multi-step sequences where fragment order risks a misread — resume terse after.

**Why:** Output was verbose despite caveman/dehumanize being loaded — full tables, bullet sections, explanatory prose for a PageSpeed report that should have been 4 lines. The skills weren't enforcing terseness; they needed a hard rule backing them.

**How to apply:** Before every response, self-check: "Would this fit in 3–5 lines without losing technical substance?" If not, cut harder. Drop articles, hedging, pleasantries, prose transitions. State findings directly.
